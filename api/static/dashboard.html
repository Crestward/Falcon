FALCON/Overview
Multi-agent fraud investigation pipeline

Pick a fraud type below to watch the agents run.

FALCON is a LangGraph supervisor that orchestrates five agents — Triage, Account Historian, Network Mapper, Pattern Hunter, Case Writer — over a Postgres + pgvector store. Each click below replays one full investigation end to end: the supervisor's event timeline, each agent's reasoning, and the final case file.

    How it works

    The supervisor at the top routes alerts through the five agents below. Click any node for the role and tools it owns.

    6 nodes

    Guardrails

    Five validators in guardrails/. Each runs at a defined choke point during an investigation. Failures are written to the security_events table.

    Metrics

    Operational data pulled from the audit tables (investigations, agent_traces, tool_call_logs). Computed on demand by GET /metrics; no Prometheus or external scraper involved.
